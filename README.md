# 不動産大家AIチーム — Realestate AI Team

> **Claude Code の中に、不動産賃貸業を支える7人のAIエージェントを住まわせるテンプレートです。**
> 解凍 → ターミナルで開く → `claude` 起動 の3ステップで、あなた専用の "AI大家チーム" が立ち上がります。

## 🎯 これは何？

不動産賃貸業は本来1人では抱えきれない仕事の集合体です。

- 物件を探して評価する（物件開拓）
- 銀行と交渉して融資を引く（融資調達）
- 入居者・修繕・管理会社をさばく（賃貸管理）
- 確定申告・経費を整理する（経理）
- 契約・トラブルに対応する（法務）
- 全体を見て次の一手を決める（戦略）

このテンプレートは、それらを **7人のAIエージェント** に分担させ、**秘書を窓口** にして相談できる仮想組織のセットです。Claude Code 上で起動するだけで、各エージェントが役割に応じた口調・判断軸で応えてくれます。

## ⚡ クイックスタート（3分で動かす）

### 前提
- [Claude Code](https://claude.com/claude-code) のデスクトップアプリがインストール済み

### 手順

1. **このリポジトリをダウンロード**
   - **Option A（簡単）**: 右上の `Code` → `Download ZIP` で ZIP を取得 → ダブルクリックで解凍
   - **Option B（更新追従したい人）**: `git clone https://github.com/Freex-Ltd/realestate-ai-team.git`

2. **Claude Code アプリで作業フォルダを開く**
   - Claude Code アプリを起動
   - 左上の「フォルダを開く」（Open Folder）ボタンをクリック
   - 解凍した `realestate-ai-team` フォルダを選択

3. **秘書に挨拶**
   - 画面下のチャット欄に次のメッセージを送る:
   ```
   こんにちは、よろしくね
   ```
   - 秘書のしおりが応答し、初回ならヒアリングを始めてくれます

> 💡 ターミナルは触らなくてOK。すべて Claude Code アプリの GUI で完結します。

### 試してみる指示

```
自己紹介して
```
→ 7人のメンバーがそれぞれ役割を名乗ります

```
property/portfolio/sample-apartment.md を見て、この物件の収支を分析して
```
→ ハンター → 軍師 → 帳簿番 と橋渡しされて、立体的な分析が返ります

```
legal/contracts/sample-rental-contract.md をレビューして
```
→ 奉行が契約書の論点を整理してくれます

## 🛠️ カスタマイズ

### あなたの情報は秘書がヒアリングしてくれる

**初回起動時、秘書しおりが対話形式であなたのプロフィールをヒアリングしてくれます。**

```
あなた: こんにちは、よろしくね
しおり: はじめまして、秘書のしおりです。チームのみんなに
        あなたのことを伝えたいので、何点かおうかがいしてもいいですか？
       （1問ずつ、優しく質問してくれる）
```

ヒアリングした内容は [`secretary/owner-profile.md`](secretary/owner-profile.md) に保存されます。後から自分で直接編集することも、「プロフィールを更新して」と話しかけて変更することも自由です。

> 💡 ルートの `CLAUDE.md` は組織のコア設定なので、基本的に触らずに済む設計です。あなたの情報は専用の `owner-profile.md` で管理されます。

### キャラ名を変える

「しおり」「ハンター」が気に入らなければ、好きな名前に変えられます。
- [`CLAUDE.md`](CLAUDE.md) と [`.claude/rules/departments.md`](.claude/rules/departments.md) の名前を一括置換してください

### 部署を追加・削除する

- **追加**: フォルダを新設 → 1枚の `CLAUDE.md` を書く → ルートの組織図に追記
- **削除**: フォルダごと削除 → ルートの組織図から該当行を消す

### 物件・銀行データを入れる

- 既存物件: `property/_template.md` をコピー → `property/portfolio/<物件名>.md` で保存
- 取引銀行: `banking/_template.md` をコピー → `banking/banks/<銀行名>.md` で保存

## ❓ FAQ

**Q. 既に自分の CLAUDE.md がある場合は？**
A. 上書きしないでください。別フォルダで起動するか、自分の CLAUDE.md に7人組の組織図セクションを追記する形がおすすめです。

**Q. Claude Code が日本語で返ってこない**
A. 各 `CLAUDE.md` に「日本語で回答してください」を冒頭に追記してください。

**Q. 入居者情報・銀行担当者名はどこに書く？**
A. `.gitignore` で `secretary/inbox/` を除外しているので、機密情報はそこに置けばGit管理から外れます。公開リポジトリにコミットする際は必ず確認を。

**Q. MCP（外部連携）は使える？**
A. このテンプレートはローカルファイル操作だけで完結する設計です。MCPを足したい場合は `.claude/` 配下にお好みの設定を追加してください。

**Q. 法人で使いたい**
A. ルート `CLAUDE.md` の「オーナープロフィール」に法人情報を書き、`finance/` の経理ルールを法人決算に合わせて調整してください。

**Q. アンインストール方法**
A. フォルダごと削除すればOKです。Claude Code アプリ本体の設定には影響しません。

**Q. ターミナルからも起動できる？**
A. はい、CLI 派の方は `cd realestate-ai-team && claude` でも動きます。本テンプレートはデスクトップアプリとCLI両対応です。

## 🔒 セキュリティ

- `.claude/settings.json` に推奨の `deny` ルール（`rm *` / `sudo *` / `curl | bash`）をプリセット済み
- 個人情報は `.gitignore` で除外される `secretary/inbox/` か `**/personal/` 配下に保存
- 公開リポジトリにフォークする場合は、`portfolio/` や `banks/` のファイルを必ず確認してから push
